# Apply migrations 0025–0029 + redeploy create-payment

These add ratings, server-side cancellation, dispatch cron, the cash-commission
cap, and the mock cashless-payment sandbox. Do them **in order** — later files
depend on functions/tables from earlier ones.

Project: `vjvointiicklyylvqzdh` (AngkorGo)

---

## A. Apply the SQL migrations (Supabase SQL Editor)

For **each** file below, in order:

1. Open **Supabase Dashboard → SQL Editor → New query**.
2. Open the local file, copy its **entire** contents, paste into the editor.
3. Click **Run**. Confirm "Success. No rows returned" (these are DDL/functions).
4. Only then move to the next file.

- [ ] **0025_ride_reviews.sql** — `trip_reviews` table, `submit_trip_review()`, `recompute_provider_rating_all()`
- [ ] **0026_ride_ops.sql** — `cancel_trip()`, `run_dispatch_sweep` cron (`ride-dispatch-sweep`), `enforce_commission_cap()` trigger
- [ ] **0027_food_ops.sql** — `order_reviews`, `submit_order_review()`, `recompute_restaurant_rating()`, `cancel_order()`, `run_food_dispatch_sweep` cron (`food-dispatch-sweep`)
- [ ] **0028_booking_cancel.sql** — `cancel_booking()`
- [ ] **0029_request_cancel.sql** — `cancel_request()`

Local path: `supabase/migrations/<file>`

### Prerequisites the cron migrations assume
0026 and 0027 call `cron.schedule(...)`, which needs the **pg_cron** extension.
If a "schema cron does not exist" error appears:

- [ ] Dashboard → **Database → Extensions** → enable **pg_cron** (and **pg_net** if a webhook call errors), then re-run that file.

## C. Smoke test (after A + B)

- [ ] **Ratings** — complete a trip → the rating card submits without error; provider rating updates.
- [ ] **Cancel** — start a ride/food/booking/request, cancel it → status flips to `cancelled`, any held payment reverses.
- [ ] **Cash cap** — a driver who owes ≥ $20 cash commission can't toggle online (blocked with a message).
- [ ] **Cron** — Dashboard → Database → **Cron Jobs** shows `ride-dispatch-sweep` and `food-dispatch-sweep` scheduled every minute.
- [ ] **Sandbox payment** — request → pay → tap **"✓ Simulate payment"** → status reaches `released`, wallet credited, 90/10 split recorded.

## D. Rollback (if a file errors mid-way)
Each file is idempotent-ish (`create or replace function`). If one fails,
fix the reported line and re-run **only that file**; already-applied earlier
files don't need re-running.
